Privacy

GlossBoss is designed to keep most work in your browser. This page summarizes the data the app stores locally and the data that is sent to backend services when you use optional cloud features.

Browser storage

  • Editor state, draft data, glossary preferences, and some UI settings are stored in your browser's local storage.
  • If you enter a personal DeepL API key, it is kept in this browser's session storage by default and cleared when you close all GlossBoss tabs. If you explicitly enable the option to remember the key, it is also stored in local storage on that browser until you clear it.

Feedback submissions

  • The feedback form sends your submission to a Supabase Edge Function protected by Cloudflare Turnstile.
  • Submitted data can include the feedback title, details, filename, browser user agent, app version, page URL, and optional contact email.
  • Feedback is converted into a GitHub issue for project triage.

Third-party services

  • DeepL is used for machine translation and glossary management when you enable translation features.
  • WordPress.org services are used to fetch glossary CSV exports, plugin metadata, and plugin source references.
  • Supabase Edge Functions proxy glossary CSV exports and plugin source lookups. The WordPress.org plugin info API is currently fetched directly from your browser because that endpoint already supports CORS and does not require server-managed secrets.

Your controls

  • You can clear saved DeepL credentials from the settings dialog.
  • You can clear glossary caches and local drafts from the app by removing browser storage for this site.
  • You can avoid entering an email address when sending feedback.

The corresponding source code for this deployment should be available from its operator.
