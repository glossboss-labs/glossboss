# Security Policy

## Reporting a vulnerability

Please do not open a public GitHub issue for security problems.

Preferred reporting path:

1. Use GitHub's private vulnerability reporting for this repository if it is enabled.
2. Email the maintainer privately at <mailto:hi@toine.zip>.
3. If email delivery fails, contact the repository owner through their GitHub profile: <https://github.com/toineenzo>.

Include:

- a description of the issue
- affected areas or files
- reproduction steps or a proof of concept
- any suggested mitigations

## Response expectations

This project is maintained on a best-effort basis, but responsible private reports are appreciated and will be reviewed before public disclosure when possible.
